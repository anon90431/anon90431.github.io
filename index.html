Identifying Knowledge Types and Developing a Reasoning Framework for Solving the Winograd Schema Challenge

Recently, there have been many works towards solving the Winograd Schema Challenge. These works mention the need of knowledge in solving the challenge but do not focus on identifying the different types of knowledge so that specific extraction techniques can be implemented to extract each such knowledge or/and type specific reasoning framework could be implemented to handle a particular type of knowledge. The major works are based on quasi-probabilistic correlation calculus, relevance theory and an ensemble of natural language processing techniques.


Though most of the works mention the requirement of knowledge in solving the problem, there has been very little work towards identification of the types of knowledge needed to correctly solve the challenge, so that different extraction intuitions/techniques can be implemented to extract each such knowledge or/and different reasoning framework could be used to handle a particular type of knowledge.


In this work, we identified 12 commonsense knowledge types such that about 98% of the Winograd Schema Challenge problems can be answered correctly by using at-least one instance of knowledge belonging one of the types. We also defined a logical reasoning framework to handle 10 out of 12 knowledge types, covering about 85% of the overall corpus. Later we provided an empirical evaluation of our reasoning framework by automatically extracting the required knowledge instances for 100 WSC problems and correctly solving them.

Reasoning Framework in Detail




Experiment Setup


The reasoning framework defined in this work relies on two assumptions, (1) a WSC problem (sentence and question) can be automatically translated into the desired formal representation, and (2) the required knowledge instances can be automatically extracted. In this section, we show the results of an empirical evaluation of the reasoning framework by automating the above two assumptions to a great extent with the help of an off the shelf semantic parsing tool and an in-house knowledge extraction system.


We used a freely available semantic parser named K-Parser (made freely available by the authors on github) for translating the input sentences and questions into the desired formal representation.


We developed a rule based knowledge extractor for this work. The basic idea of knowledge extraction is inspired from the knowledge hunting work mentioned in the IJCAI 2015 paper on Winograd Schema Challenge. There are aspects of our system,

  • creating search queries from the WSC problems
  • using a search engine to collect the sentences which satisfy the queries
  • extract knowledge from the collected sentences, in case they contain any

Because of the limited availability of search engine access and the main focus of this work not being the automatic knowledge extraction, the first two aspects were carried out manually. The collected sentences are then passed one by one to an in-house rule based knowledge extraction module. The module uses semantics (from K-Parser) of the input sentence to find the patterns which satisfy the knowledge types (Type 1 to 10) handled by our reasoning framework.
